# 02 / 方法与路线图

## 为什么这个任务要按“长期研究”来做

原因很简单：

1. **来源分散**：鸭老师的相关内容分布在不同主题帖、分页、评论链和外部讨论里。
2. **行业变化快**：AI漫剧的工具、平台、监管、分成模式都在快速变化。
3. **同一句话的含义会变**：比如“平台真给钱”，在不同月份、不同政策环境下，含义可能完全不同。
4. **单次抓取不够**：summary 页在当前环境下无法直接抓到完整内容，只能逐步补强。

所以正确做法不是“一次性写完”，而是：

> 先搭好可扩展的证据框架，再不断往里加高置信内容。

## 研究问题拆分

本项目把“鸭老师怎么看 AI漫剧”拆成六个子问题：

### 1. 市场判断
- 他认为 AI漫剧是不是风口？
- 风口是真风口，还是补贴驱动型机会？
- 他如何区分“能赚钱”和“神话叙事”？

### 2. 内容形式判断
- AI视频 vs 动态漫画，他更偏哪种？
- 为什么切换形式？
- 形式切换背后的关键变量是什么？

### 3. 制作与交付
- 他怎么看人物一致性、抽卡、返修、节奏、起量测试？
- 他更像在讲创作，还是在讲生产管理？

### 4. 平台与分发
- 他如何理解推流？
- 他对起号、平台规则、分发适配的关注点是什么？

### 5. 商业与承制
- 他怎么看报价、周期、外包、团队协作、平台合作、IP授权？
- 他的风险意识主要落在哪些环节？

### 6. 方法论演化
- 早期的判断和后续是否发生变化？
- 是不是从“技术试验”走向“工业化试产”？

## 来源分层方法

### 一级来源：原始公开内容
优先级最高。

包括：

- Linux.do 原始主题帖
- Linux.do 分页页
- 用户页 / 用户可访问摘要页

用途：

- 归纳鸭老师本人观点
- 记录明确可追溯发言
- 建立时间线

### 二级来源：行业上下文
包括：

- 行业报道
- 平台政策整理
- 媒体采访 / 数据稿
- 研究/评论文章

用途：

- 对照鸭老师的判断是否保守 / 激进 / 反主流
- 给“平台真给钱”“一人制片不现实”等说法提供背景

### 三级来源：镜像与检索线索
包括：

- Telegram 频道镜像
- 聚合站摘要
- 搜索结果摘要

用途：

- 找入口
- 保留检索痕迹
- 发现遗漏帖子

注意：

- 三级来源不能单独支撑高置信结论
- 它们只能作为“后续回查”的线索层

## 抓取与整理规则

### 规则 1：尽量保存最小可验证单元
每条材料至少记录：

- `source_id`
- `url`
- `title`
- `capture_method`
- `excerpt`
- `confidence`
- `note`

### 规则 2：分析和证据分开写
- 证据 → `data/source-captures/*.json` + `docs/03-source-register.md`
- 判断 → `docs/01-initial-analysis.md`

避免把“我的推断”写成“来源原文”。

### 规则 3：把“是谁说的”看得比“说了什么”更重要
对于论坛这种多用户混杂讨论：

- 若能确认是鸭老师本人，标为高优先
- 若只是其他人评价他，单独标为背景/回声
- 若归属不清，只留作线索

### 规则 4：允许不完整，但不允许假装完整
当前环境下 summary 页面受限，就写清楚受限；

不能把搜索摘要包装成“我已完整通读其主页”。

## 版本维护节奏

建议节奏：

### 每次增量更新都做
- 更新来源登记
- 更新 source capture JSON
- 记录新增主题 / 分页 / 线索

### 每 3~5 次增量更新再做
- 重新梳理分析页
- 调整结论置信度
- 去除已经失效或证据太弱的判断

### 每月做一次
- 输出版本快照
- 增加一段“本月变化”
- 检查平台政策/监管是否影响旧结论

## 当前最重要的操作准则

### 准则 A：优先保真，不优先堆量
宁可少写 10 条，也不要把不清楚归属的材料硬塞进结论。

### 准则 B：优先抓“方法论话语”
优先收集这类内容：

- 为什么换形式
- 为什么拒绝项目
- 为什么说平台真给钱但不等于暴富
- 为什么强调返修 / 外包 / 批量交付

这类内容比单纯“秀作品”更能揭示底层认知。

### 准则 C：优先找“变化”
静态总结很容易过时。
真正有价值的是：

- 他原来怎么想
- 后来为什么变了
- 哪些变化来自平台，哪些来自生产现实

## 当前环境限制记录

### 已知限制
- 直接抓取 `https://linux.do/u/yalaoshi/summary` 遇到 Cloudflare 拦截
- 某些论坛分页只能拿到搜索摘要
- 某些外部结果可能是转载/再包装，而非论坛原帖

### 对策
- 先保存可回溯 URL 和摘要
- 结论按置信度分级
- 后续有更高保真来源时再替换

## 最终目标不是“做全”，而是“做得越来越准”

这个项目真正要积累的，不只是资料数量，而是：

- 证据质量
- 判断边界感
- 版本可追溯性
- 后续可接手性
