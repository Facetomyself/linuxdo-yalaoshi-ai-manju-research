## 工作原则

### 1. 来源优先级
按以下顺序取证：

1. Linux.do 原始主题页 / 分页页
2. Linux.do 用户页或可验证的用户索引页
3. 外部媒体文章（只作行业背景对照）
4. Telegram 镜像 / 聚合页 / 摘要站（只作线索）

### 2. 结论分级
每条结论都尽量打标签：

- `高置信`：有明确原始页面，且归属清楚
- `中置信`：有摘要或分页片段，归属大概率正确，但待补全文
- `低置信/线索`：只能当入口，不能直接拿来下重结论

### 3. 记录方式
新增来源时，至少同步更新：

- `data/source-captures/...json`
- `docs/03-source-register.md`
- 如能影响结论，再更新 `docs/01-initial-analysis.md`

### 4. 引用方式
尽量使用：

- 短摘录
- 自己总结
- 明确 URL
- 明确来源类型

避免：

- 大段复制原文
- 没有上下文的强行解读
- 把论坛回复的情绪话当方法论

## 成功标准
短期成功不是“全部写完”，而是做到：

1. 新人进 repo 后 3 分钟内知道这个项目在研究什么
2. 每条核心判断都能回到来源入口
3. 后续任何一次增量更新都能接着做，而不是重来
4. 研究结论逐渐从“零碎摘要”过渡到“高置信方法论”
